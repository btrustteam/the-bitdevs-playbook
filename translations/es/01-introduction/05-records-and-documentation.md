# Registros y Documentación

A cada sede de BitDevs se le asigna una carpeta compartida con el nombre de su ciudad (por ejemplo, BitDevs Nairobi) para almacenar reportes, presupuestos y registros financieros. Esta carpeta es gestionada por el equipo organizador y debe mantenerse actualizada y bien organizada, ya que es revisada mensualmente por Btrust.

## Estructura de Carpetas

La carpeta principal de cada sede debe incluir las siguientes dos subcarpetas fundamentales:

### 1. Reportes de Meetups

Esta carpeta contiene la documentación y resúmenes de cada meetup mensual. Cada entrada debe incluir:

- Un reporte de evento completado usando la plantilla estándar *(ver [Anexo C: Formato de Reporte de Evento](../08-annexes.md))*
- Conclusiones clave de la sesión
- Fotos, si están disponibles y compartidas con consentimiento
- Agenda, lista de ponentes y datos destacados de asistencia
- Un enlace a la carpeta de recibos y cualquier documento de respaldo relevante
- Notas sobre desafíos enfrentados o lecciones aprendidas
- Acciones de seguimiento y planes para el próximo meetup

**Convención de nombres:**
`YYYY-MM_BitDevs XXX_Meetup_Report`
*(por ejemplo, 2025-04_BitDevs Nairobi_Meetup_Report)*

### 2. Finanzas

Esta carpeta se utiliza para registrar toda la actividad financiera relacionada con las operaciones del meetup. Cada entrada debe incluir:

- Un reporte financiero completado usando la plantilla estándar *(ver [Anexo D: Formato de Reporte Financiero](../08-annexes.md))*
- Recibos y facturas escaneadas por cada gasto
  - Estos deben subirse en una carpeta separada y vincularse en el reporte
- La dirección BTC utilizada para desembolsos o reembolsos
- Cualquier nota especial, aprobación o contexto para reembolsos o costos extraordinarios

**Convención de nombres:**
`YYYY-MM_BitDevs XXX_Finance_Report`
*(por ejemplo, 2025-04_BitDevs Nairobi_Finance_Report)*

## Plazo de Entrega

Toda la documentación de meetups y finanzas debe subirse antes del **10 de cada mes**, cubriendo el meetup del mes anterior.

### Protocolo de Archivado y Acceso

Cada equipo organizador es responsable de mantener su carpeta actualizada y precisa.

- El acceso está restringido a los organizadores de la sede y personal autorizado de Btrust
- Los archivos **nunca deben eliminarse**
- Los archivos más antiguos deben moverse a una subcarpeta llamada: `Archive_Year`

## Cumplimiento

La documentación consistente y precisa es un requisito para mantener el estatus de patrocinio. Las sedes que no actualicen sus registros regularmente pueden experimentar retrasos en el financiamiento o suspensión del apoyo.
